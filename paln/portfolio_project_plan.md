# 個人作品集網站規劃書

本規劃書為 Hana 打造一頁式個人作品集網站的整體藍圖，結合了日系小清新與現代文青極簡的視覺風格，並針對前端開發、SEO 及資訊架構進行全面規劃。

## 一、 網站基本資訊 (General Information)
- **網站擁有人：** Hana
- **網站型態：** RWD 響應式一頁式網站 (One-Page Portfolio)
- **技術目標：** 極致效能、語意化 HTML 結構、SEO 友善

## 二、 網站核心功能與區塊架構
本網站將分為六大核心區塊，由上至下依序排列：

| 區塊名稱 | 建議內容 | 核心 HTML 元素 |
| :--- | :--- | :--- |
| **1. 導覽列 (Nav)** | 個人 Logo/名稱、錨點連結、GitHub 與 LinkedIn 圖示。固定於頂部（Fixed）。 | `<header>`, `<nav>`, `<ul>`, `<a>` |
| **2. 英雄區 (Hero)** | 強烈核心標語（如：專注效能與細節的全端工程師）、一兩句簡短定位描述、清晰個人風格照、CTA 引導按鈕。 | `<section>`, `<h1>`, `<p>`, `<button>` |
| **3. 關於我 (About)** | 個人故事、為什麼喜愛開發、學習特質。條列式展示目前核心技術棧（C#, SQL Server, HTML/CSS 等）。 | `<section>`, `<h2>`, `<p>`, `<ul>`, `<li>` |
| **4. 精選作品 (Portfolio)** | 文青極簡風專案卡片（包含圖片、專案名稱、技術標籤、簡述、GitHub 與 Demo 網址）。預計展示 2~3 個核心專案。 | `<section>`, `<article>`, `<h3>`, `<img>`, `<span>` |
| **5. 聯絡方式 (Contact)** | 歡迎聯繫的邀請文字、主動喚起信箱的 `mailto:` 連結、LinkedIn 與 GitHub 快速轉跳連結。 | `<section>`, `<h2>`, `<a>` |
| **6. 頁尾 (Footer)** | 版權所有聲明（© 2026 Hana. All rights reserved.）。 | `<footer>`, `<small>` |

## 三、 前端開發策略與技術亮點
- **切版策略：** 建議使用 Mobile-First（行動裝置優先），主要佈局依賴 CSS Flexbox 與 CSS Grid 確保在不同螢幕尺寸（RWD）皆能完美契合。
- **效能管理：** 嚴控圖片檔案大小，作品截圖必須優化並轉換為 `.webp` 或 `.avif` 格式，並對 Hero 區以外的圖片套用 `loading="lazy"` 以提升載入速度。

## 四、 本地端與搜尋引擎優化 (SEO)
- **語意化權重：** 嚴格控管整個頁面只有一個唯一的 `<h1>`（位於 Hero 區），其餘大標題統一使用 `<h2>`。
- **結構化資料：** 網頁底層建議埋入 JSON-LD 格式的 `Person` Schema，協助 Google 爬蟲精確識別此網站為 Hana 的個人官方專頁。
